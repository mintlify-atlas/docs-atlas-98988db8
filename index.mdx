## The Problem

When an order is marked as "SHIPPED" in an e-commerce system, multiple actions need to occur:

- Send a tracking email to the customer
- Log the event for audit purposes
- Update statistics counters

Without the Observer pattern, the `OrderService` would need to know about all these different systems, creating tight coupling and making the code difficult to extend.

## The Solution

This implementation uses the Observer pattern to:

1. **Define a Subject interface** (`OrderSubject`) that manages observers
2. **Define an Observer interface** (`OrderObserver`) for notifications
3. **Implement concrete observers** for email, logging, and statistics
4. **Decouple the service** from notification details
